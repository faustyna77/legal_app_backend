
# Legal RAG API

Semantic search engine for Polish court judgments and legal acts.

## Base URL

```
http://127.0.0.1:8000
```

Swagger UI: `http://127.0.0.1:8000/docs`

---

## Authentication

All protected endpoints require an API key in the request header:

```
x-internal-key: <INTERNAL_API_KEY>
```

**In Swagger UI:**
1. Click the **"Authorize"** button (top right)
2. Enter the API key in the **"Value"** field
3. Click **"Authorize"** → **"Close"**
4. All subsequent requests will be authenticated automatically

## Endpoints

### Search

#### `POST /search`
Semantic search across all judgments and legal acts.

**Protected:** Yes

**Request body:**
```json
{
  "query": "zasiłek chorobowy emerytura",
  "filters": {
    "court": "Sąd Rejonowy Szczecin-Centrum w Szczecinie",
    "city": "Szczecin",
    "legal_area": "Prawo pracy",
    "source": "saos",
    "date_from": "2022-01-01",
    "date_to": "2023-12-31",
    "court_type": "Sąd powszechny"
  }
}
```

All filters are optional. Use `"filters": {}` for no filtering.

**Response:**
```json
{
  "answer": "LLM-generated answer based on retrieved documents",
  "sources": [...],
  "judgments": [...],
  "articles": [...],
  "latency_ms": 4800
}
```

### Judgments

#### `GET /judgments`
List judgments with optional filtering and pagination.

**Protected:** No

**Query parameters:** `court`, `date_from`, `date_to`, `limit` (max 100), `offset`

---

#### `GET /judgments/{id}`
Get full details of a single judgment.

**Protected:** No

---

#### `GET /judgments/{id}/similar`
Find similar judgments based on vector similarity.

**Protected:** No

**Query parameters:** `limit` (max 20)

---

#### `GET /judgments/{id}/summary`
Generate or retrieve a structured summary of a judgment.

**Protected:** Yes

**Response:**
```json
{
  "id": 1,
  "case_number": "IX P 440/21",
  "court": "Sąd Rejonowy Szczecin-Centrum w Szczecinie",
  "date": "2023-01-02",
  "summary": {
    "teza": "...",
    "stan_faktyczny": "...",
    "rozstrzygniecie": "...",
    "podstawa_prawna": "..."
  },
  "cached": true
}
```

> `cached: true` means the summary was retrieved from the database (fast). `cached: false` means it was generated by the LLM now (~4-5 sec) and saved for future calls.

---

#### `POST /judgments/{id}/chat`
Ask a question about a specific judgment. The LLM answers based exclusively on that judgment's content.

**Protected:** Yes

**Request body:**
```json
{
  "question": "Czy powódce przysługuje odprawa emerytalna?"
}
```

**Response:**
```json
{
  "judgment_id": 1,
  "case_number": "IX P 440/21",
  "question": "...",
  "answer": "LLM answer based on judgment content",
  "chunks_used": 5,
  "chunks": [
    {
      "content": "fragment of judgment text...",
      "similarity": 0.5478
    }
  ]
}
```

### Filters

#### `GET /filters`
Returns all available filter values.

#### `GET /filters/courts`
List of all courts in the database.

#### `GET /filters/cities`
List of all cities.

#### `GET /filters/legal-areas`
List of all legal areas.

#### `GET /filters/sources`
List of sources (`saos`, `nsa`).

#### `GET /filters/court-types`
List of court types.

---

### Health

#### `GET /health`
Check if the API is running.

```json
{"status": "ok", "version": "1.0.0"}
```

## Tech Stack

| Component | Technology |
|-----------|------------|
| Backend | FastAPI (Python 3.12) |
| Database | PostgreSQL + pgvector (NeonDB) |
| Embeddings | Jina AI `jina-embeddings-v3` (1024-dim, multilingual) |
| LLM | Groq `llama-3.3-70b-versatile` |
| Data sources | SAOS API, NSA scraper |
| Search | Semantic RAG — cosine similarity on 1024-dim vectors |

---

## Database

| Table | Description |
|-------|-------------|
| `judgments` | Court judgments with embeddings and summaries |
| `judgment_chunks` | Judgment fragments with embeddings for RAG |
| `legal_acts` | Legal acts metadata |
| `articles` | Articles from legal acts |
| `users` | Users (auth planned) |
